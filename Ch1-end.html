小结

通过前面几节的学习，我们掌握了命令提示符的相关操作、命令及其分类、环境变量 path 和 pathext 的意义。


命令提示符

又名控制台，在 类 UNIX 操作系统 中习惯上叫做终端。 它是批量处理和输出信息的主要平台，所以，必须要掌握它。 

	打开命令提示符

鼠标方法一：开始 -> 所有程序 -> 附件 -> 命令提示符
鼠标方法二：我的电脑/计算机 -> C: -> Windows -> System32 -> cmd.exe

键盘方法：Win+r -> 输入 cmd 回车
衍生方法：开始 -> 运行 -> 输入 cmd 回车，或者点击确定


	创建批处理脚本文件

在桌面鼠标右键 -> 新建 -> 文本文档 -> 重命名为 test.bat ->
	还需要两次确认才标志着 成功创建 一个空的批处理脚本文件
鼠标右键选择创建成功的文件 -> 编辑 -> 使用记事本编辑脚本

或者：Win+r -> notepad test.bat 回车 -> 确定 创建文件


	终止脚本

键盘： Ctrl+c 强制中断批处理，在询问之后输入 y 并以回车结束就可以退出了
鼠标： 直接点击命令提示符右上角的关闭按钮，但是很多大问题的时候不太好用
如果你需要退出命令提示符，这个命令自己研究研究吧 
> exit /?


path 与 pathext

环境变量是很特殊的东东，path 和 pathext 是全局有效地。 关于变量的内容斑驳会在后面的章节中慢慢介绍，我们先来回顾一下 path 以及 pathext 吧。

path 指定了系统搜索外部命令的路径集合，默认 C:\Windows\System32\ 的查找优先级最高。 如果没有 path 的支持，我们就只能使用内部命令以及当前目录下的命令了 (如果命令中包含某个外部命令的绝对路径，也是可以调用的)。 所以，如果没有必要，不要去删除之前已经存在的路径。

pathext 标志着默认的可执行文件的后缀名。如果你能想的多一点，这个地方可以做很多事情的，(*^__^*) 嘻嘻……


系统配置

除了查看系统版本、硬件信息之外，Windows 也是可以自定义修改一部分数据的。 整天对着 XP 的主题，佛祖看着也会感到乏味的，O(∩_∩)O哈哈~ 关于文件夹选项的操作，之前也有部分介绍， 里面还有很多系统相关的配置，推荐读者去做一番研究，会有很多收获哦， 不过它们的实质都是在对注册表进行修改。
